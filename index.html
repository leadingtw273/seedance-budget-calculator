02

計費公式管線

官方表單背後的實際運算順序
1
輸出尺寸由「解析度 × 比例」決定
W, H ← lookup(Resolution, Aspect Ratio)
每個解析度的總像素量固定，依比例分配長寬（見 §4）。FPS 永遠是 24。
2
每秒 token 係數
factor = W × H × FPS ÷ 1024
這是「每一秒影片」消耗的 token 數，與模型無關，只看像素與幀率。
3
預估 token（依實際總時長）
est = (InputDur + OutputDur) × factor　·　無輸入影片時 = OutputDur × factor
輸入影片也要計費（需重新編碼），因此計入總時長。
4
最低計費時長（僅「含輸入影片」時生效）
mindur = 2 × OutputDur − ⌊OutputDur ÷ 3⌋
含輸入影片時有最低消耗門檻，只跟輸出長度有關，與輸入長度無關。例：out=5→9s、out=10→17s、out=15→25s。最低 token = mindur × factor。
5
計費時長 = 實際與最低取大者
Counted = max(InputDur + OutputDur, mindur)　·　無輸入影片時 = OutputDur
計費 token = Counted × factor＝ max(預估, 最低)。短輸入＋長輸出時，最低門檻常常蓋過實際時長。
6
最終價格
Price (USD) = 計費 token × 單價 ÷ 1,000,000
單價隨「模型 × 解析度 × 是否含輸入影片」變動（見 §3）。實際消耗以 API 回傳的 usage.completion_tokens 為準。
